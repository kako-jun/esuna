# Esuna — ロードマップ

## 現在の状態

基本的なWebアプリの骨格が完成している。9マスグリッドUIの枠組み、はてなブックマーク・5ch・SNSのコンテンツ取得、テキスト読み上げの基礎実装が揃っている段階。

---

## 近い将来に追加したいもの

### メモアプリ

視覚障害者にとって、考えをまとめることはとりわけ難しい。どこまで考えたか、どんな結論を出したか、記録していかないと流れてしまう。

Esuna内で完結するシンプルなメモ機能を追加する。音声入力でメモを作り、テキスト読み上げで後から確認できる。整理には9マスUIを使い、カテゴリ分けも直感的に行える。

### 音楽プレイヤー

見なくても楽しめるコンテンツとして、音楽は自然な選択だ。ローカルの音楽ファイルを読み込み、テンキーで再生・停止・曲送りを操作する。再生中の曲名・アーティスト名を読み上げる。シャッフル・リピートも9マスの操作で。

### ゲーム

AudioGames.net のようなポータルが存在するように、視覚なしで楽しめるゲームには需要がある。Esuna内で動く音声中心のゲームを追加していく。

最初の候補はテキストアドベンチャーやサウンドノベル。プレイヤーの選択を読み上げ、結果を読み上げる。ゲームの進行状況もEsunaのメモ機能と連携して記録できるとよい。

RPG要素を持つゲームも視野に入れている。ドラクエのようにウィンドウの変化をナレーションで伝える仕組みを参考にして、戦闘・移動・会話をすべて音声だけで楽しめるゲームを設計したい。

## 中期的に取り組みたいもの

### AI視覚アシスタント

「この画面に何が写っていますか」「この写真の人は何をしていますか」という疑問に、AIが答えてくれる機能。

ローカルで動くマルチモーダルモデルを組み込み、カメラや画像ファイルを渡すと内容を説明してくれる。外出先でレシートを読み上げてほしい、郵便物の宛名を確認したい、といった実用的な場面を想定している。

サーバーに送らず端末内で処理することで、プライバシーを守りながら使える。

### Esuna Board（質問掲示板）

視覚障害者が困ったときにすぐ助けを求められる、チャット形式の軽量掲示板。

「スマホの設定画面の左から3番目のボタンは何ですか」のような、目の見える人ならすぐわかることを気軽に聞ける場所。投稿は自動的に消えていくためプライバシーが守られる。匿名で使える。

「補助キボンさん」（教えてほしい側）と「優しい名無しさん」（教えてくれる側）の2ロールで設計。マスコットはS菜さん（目を閉じた中性的な少女）。

socket.io を使ったリアルタイム通信で、待ち時間を感じさせない体験を目指す。

## 長期的な構想

### 小説リーダー・ノベルゲーム

青空文庫のような公開テキストをEsunaで読む機能。ノベルゲームの会話シーンのように、ナレーターとキャラクターで声を変えて読み上げるとより楽しい体験になる。

### 意思伝達支援

言葉にしにくい状況や、声が出しにくい場面で、考えていることを伝えるための機能。定型文の選択、絵カードによる意思表示など、コミュニケーション支援への応用も考えている。

### 多言語・国際対応

Esunaを使いたいのは日本語ユーザーだけではない。読み上げ言語の切り替え、コンテンツソースの地域設定、スクリーンリーダーのローカライズ対応を整えることで、世界中のユーザーに届けられる。

---

## 大切にしたいこと

機能を増やすことよりも、今ある機能が確実に動くことの方が重要だ。

視覚障害を持つユーザーが「動かなかった」という経験をすると、次に開く気が失せる。信頼を積み上げるために、地道に品質を高めていく。

## 既知のバグ

### 高優先度
- [ ] おまかせモード実行中に戻ると、次回起動時にエラー
- [ ] 音声メモが100件を超えるとLocalStorageエラー（エラーハンドリング不足）
- [ ] ラジオ再生中に他のページに移動すると音声が止まらない

### 中優先度
- [ ] 長文の読み上げ中にEscapeを押しても止まらないことがある
- [ ] Podcast再生時、エピソード終了後の自動遷移がない
- [ ] RSSフィードの取得に失敗してもエラー表示がない

### 低優先度
- [ ] メインメニューの音声が長すぎる
- [ ] お気に入り一覧が空の時のメッセージがない
- [ ] タイマー完了音が小さい
