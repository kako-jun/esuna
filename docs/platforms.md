# Esuna — 対応プラットフォーム

## 基本方針

Esunaはブラウザで動くWebアプリとして設計されている。これは意図的な選択だ。

インストール不要、アップデート自動、どのOSでも同じように動く。視覚障害を持つユーザーにとって、アプリのインストールやアップデートは補助なしには難しい作業だ。URLを開くだけで使えるWebアプリという形式が、最もアクセスしやすい。

## ブラウザ対応

| ブラウザ | 対応状況 | 備考 |
|---------|---------|------|
| Chrome（最新版） | 推奨 | Web Speech APIの実装が最も安定 |
| Edge（最新版） | 対応 | Chromiumベースのため動作良好 |
| Firefox（最新版） | 対応 | 読み上げ音声の種類が少ない場合あり |
| Safari（最新版） | 対応 | iOS/iPadOS での使用を想定 |
| Android Chrome | 対応 | スマートフォンでの利用に対応 |

動作に必要な主な機能：

- **Web Speech API**（読み上げに使用）
- **Fullscreen API**（全画面表示に使用）
- **Fetch API**（コンテンツ取得に使用）

古いブラウザや機能が制限された環境では、読み上げが動作しない場合がある。その場合でも、キーボード操作とスクリーンリーダーの組み合わせで利用できるよう設計している。

## スクリーンリーダー互換性

EsunaはOSのスクリーンリーダーと**共存**して動作できる。ただし二重読み上げを避けるため、Esunaを使う場合はOSのスクリーンリーダーをオフにするか、Esunaの読み上げをオフにするかを選んでほしい。

### Windows

**NVDA（NonVisual Desktop Access）**
オープンソースのスクリーンリーダー。操作がVimに似た感覚があり、キーボードに慣れたユーザーに向いている。Esunaとの相性が良く、フォーカス移動時にエリアの情報を読み上げる。

**ナレーター**
Windowsに標準搭載。追加インストール不要で使えるため、最初の一歩として利用しやすい。

### macOS / iOS / iPadOS

**VoiceOver**
Appleデバイスに標準搭載のスクリーンリーダー。macOSとiOSで動作する。EsunaはVoiceOverのフォーカス管理に対応したHTMLセマンティクスを使用しており、正しくエリア情報が伝わる。

### Android

**TalkBack**
Androidに標準搭載。スワイプジェスチャーで要素を移動する。EsunaのタッチエリアはTalkBackのジェスチャー操作とも組み合わせて使える。

### Linux

**Orca**
GNOMEデスクトップ環境向けのスクリーンリーダー。Linuxユーザーは少数派だが、技術的な理由でLinuxを使わざるを得ないユーザーも想定している。

## アクセシビリティ対応の実装方針

- `aria-label`、`role`、`aria-live` などのWAI-ARIA属性を適切に使用
- フォーカス管理を明示的に行い、キーボードだけで全機能を利用可能にする
- コントラスト比はWCAG 2.1 AA基準以上を維持
- アニメーション・点滅要素は使わない（てんかん等への配慮）
- 文字サイズは固定せず、ブラウザの設定に従う

Esunaを使うすべての人が、補助なしで操作できることを目指している。
