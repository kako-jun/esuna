# Esuna 状態一覧

最終更新: 2026-05-08

このファイルは、Esuna の進捗を `雰囲気` ではなく `分母から` 管理するための正本。
先に **何を作るつもりなのか** を固定し、そのあとで **技術的に成立するか / セキュリティや認証の前提があるか / 実装できているか** を別々に書く。

## まず結論

- いまの Esuna は「UI の試作」と「個別機能の成立検証」が混ざっている
- そのため、`未実装` と `実装途中` と `そもそも成立性未検証` が同じように見えていた
- この表ではそれを分離する
- **Issue 一覧を作る前に、分母と検証段階を固定する**

## 管理ルール

| 項目 | 意味 |
| --- | --- |
| `対象` | Esuna が提供すると言っている機能の分母 |
| `成立性` | 技術的に実現できる見込みを確認したか |
| `認証` | 個人アカウントやログインが必要か |
| `セキュリティ / 規約` | 実装してよいか、扱ってよい情報か |
| `実装状態` | 実際にどこまで作ってあるか |
| `Issue化` | いま具体作業に落とせるか |

## 判定語

### 成立性

| 値 | 意味 |
| --- | --- |
| `未検証` | 実現可能性をまだ調べていない |
| `要再設計` | 既存方式では成立していない |
| `成立見込み` | 方式は見えているが詰め切っていない |
| `成立確認` | 最低限の技術成立は確認した |

### セキュリティ / 規約

| 値 | 意味 |
| --- | --- |
| `未確認` | 規約・認証・情報取扱いを未確認 |
| `注意` | 実装可能でも、規約や運用条件に注意が必要 |
| `問題小` | ローカル機能または公開情報中心で大きな懸念が少ない |

### 実装状態

| 値 | 意味 |
| --- | --- |
| `未実装` | 画面や導線があっても、機能として成立していない |
| `試験実装` | 一部表示や取得はできるが、常用前提ではない |
| `実装途中` | 主要導線はあるが、未対応条件や失敗条件が大きい |
| `実装済み` | ローカル機能として概ね成立している |

## いま Issue を切ってよいもの

- UI 文言の整理
- 初回チュートリアル
- 弱視向けフォント
- 3x3 グリッドの視認性と操作性
- RSS のバックエンド統一
- Podcast の失敗時案内
- 青空文庫の取得失敗表示
- NHK らじるらじる再生改善

## いま Issue に落とす前に、先に成立性検証が要るもの

- X / Twitter
- radiko
- 5ちゃんねる スレッド一覧
- 5ちゃんねる レス取得

ここは「実装タスク」に分解する前に、まず次を検証する。

1. 技術的に取得方式が成立するか
2. 認証や地域制限をどう扱うか
3. 規約や運用面で継続可能か
4. Esuna のバックエンドに置くべきか

## いまの進捗の読み方

- `実装済み` が並ぶものは、主にローカル機能
- 外部サービス連携は、ほとんどが `試験実装` か `実装途中`
- `未実装` の中でも、`要再設計` と `未検証` は別物
- `未検証` を残したまま Issue を量産しない

## メモ

- 「画面がある」ことと「サービスが成立している」ことは別
- 「未実装」と「成立性未検証」は別
- ユーザーに見せる文言は、この表の判定と一致させる
