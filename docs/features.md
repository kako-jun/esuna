# Esuna — 機能一覧

## 9マスグリッドUI

画面を3行×3列の9つのエリアに分割する、Esunaの核心となる操作体系。

```
┌─────┬──────────────┬─────┐
│  1  │      2      │  3  │
│戻る │  リロード    │設定 │
├─────┼──────────────┼─────┤
│  4  │      5      │  6  │
│前へ │メインコンテンツ│次へ │
├─────┼──────────────┼─────┤
│  7  │      8      │  9  │
│機能 │    機能     │機能 │
└─────┴──────────────┴─────┘
```

- 画面の左上エリアに触れると「戻る」、中央に触れると「メインコンテンツ」
- 大きなエリアに分割されているため、タッチ位置が多少ずれても意図どおりに操作できる
- すべての画面でこのレイアウトが統一されており、遷移しても戸惑わない
- フルスクリーン表示を前提としており、OS画面に迷い込む心配がない

## テキスト読み上げ

ブラウザ内蔵の Web Speech API を使い、画面上のすべてのコンテンツを音声で届ける。

- 記事タイトル・本文を自動で読み上げ
- エリアに触れたとき、そのエリアの内容と操作説明をアナウンス
- 遷移のたびに現在地と利用可能な操作を伝える
- 読み上げ速度・音量の調整に対応
- Escキーまたは特定のエリアで読み上げを即時停止できる

現在の `停止` は `一時停止` ではなく、現在の発話と後続キューを破棄する `中断` である。
停止した本文を続きから再開する機能は、まだない。

また、自動ナビゲーションが有効な画面では `停止` と `読み終わり` の扱いがまだ整理途中であり、操作仕様として固定しきれていない。

外部の読み上げソフト（NVDA、VoiceOverなど）との二重読み上げを避けるため、OSの読み上げ機能をオフにしたままでも完全に使えるよう設計している。

## キーボード・テンキー操作

タッチ操作が難しい場面でも、キーボードだけで完全に操作できる。

- 数字キー 1〜9：対応するエリアを直接選択
- 矢印キー：エリア間を順番に移動
- Enterキー：選択・決定
- Escapeキー：読み上げ停止 / 前の画面に戻る

テンキーを使えば、指を置いた位置から番号を叩くだけで操作できる。ホームポジションを崩さずに使えることを意識した設計だ。

## はてなブックマーク閲覧

はてなブックマークの人気エントリーと新着エントリーを取得して読み上げる。

- 話題の記事タイトルを順番に紹介
- 気になった記事の本文とブックマークコメントを取得
- 外部リンクに飛ぶことなく、Esuna内で内容を確認

## 5ch閲覧（Esuna Viewer）

5ちゃんねるのスレッドを、読み上げに最適化した形で提供する。

- 板の一覧からスレッドを選択
- レス番号「>>1」を「1へ」のように自然な日本語に変換
- 投稿日時・名前欄・AAなど、読み上げの邪魔になる要素を除去
- 人気順・新着順で並び替え可能
- 閲覧専用（投稿機能は持たない）

## SNS集約

X（Twitter）・Mastodon・Bluesky の投稿を一箇所で確認できる。

- 主要SNSのタイムラインをまとめて取得
- 投稿内容を順番に読み上げ
- 外部APIへの依存を最小化し、スクレイピングで無料で動作

## フルスクリーン表示

Esunaは常に全画面で起動することを前提としている。

- 起動時に自動的に最大化
- OS画面や他のアプリへの意図しない遷移を防ぐ
- 「Esunaの中にいる」という感覚が、安心した操作体験をつくる
